# Security Policy

## Scope

This repository is a public reference library. It is not a supported package offering.

## Reporting

- Do not post secrets, tokens, or private infrastructure details in a public issue.
- Use GitHub private vulnerability reporting if available.
- Otherwise, contact the repository owner through GitHub first.

## What To Report

- accidentally committed credentials
- unsafe defaults in examples or configuration
- obvious flaws that could mislead someone reusing the patterns

## Response Model

- best effort only
- no SLA
- no guaranteed fix timeline
